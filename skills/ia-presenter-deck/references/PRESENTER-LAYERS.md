# Presenter Layers

This is the core mental model behind iA Presenter.

Treat a deck as separate layers:
- script
- structure
- visible slide surface
- design
- action / rehearsal

Weak decks usually mix all five at once.

## 1. Script

Start with the message.

Write:
- a rough script
- an outline
- pasted source text

Do not start with:
- theme choice
- image search
- polished layout

## 2. Structure

Use `---` to split one idea from the next.

Ask:
- what deserves its own slide
- what should stay grouped
- what should move earlier or later

## 3. Visible Slide Surface

By default, much of the writing is for the presenter, not the audience.

Visible by default:
- headings
- images
- videos
- tables

Visible only when explicitly promoted:
- body text with a TAB
- lists with a TAB
- block quotes with a TAB

This means you should choose carefully what the audience sees.

## 4. Design

Design comes late.

Only after the message and structure work:
- pick or refine theme direction
- replace placeholders
- tune image position and scale

Do not use styling decisions to solve story problems.

## 5. Action

The deck should support:
- rehearsal
- live delivery
- export

So ask:
- can the presenter actually speak from this?
- do the slides create curiosity instead of duplication?
- will the deck still make sense on a smaller screen?

## Working Rule

When a draft feels weak, ask which layer is actually broken.

Usually:
- a weak script is not fixed by design
- a weak structure is not fixed by more bullets
- a weak slide surface is not fixed by better notes alone
