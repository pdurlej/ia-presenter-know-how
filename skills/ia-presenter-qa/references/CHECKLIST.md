# iA Presenter QA Checklist

## Rendering Errors

Flag immediately:
- metadata rendered as visible text
- captions appearing by accident
- content blocks creating unintended extra columns
- images that push the argument off the slide
- a slide whose meaning only becomes clear when you read the notes

## Composition

Look for:
- image and text fighting for attention
- one side feeling empty and the other overloaded
- visible text stranded far from the image it belongs with
- accidental three-column layouts
- titles stranded in a left column while proof or tables drift to the right
- slides that clearly want a full-width proof layout but render as an accidental split
- image slides that want a real half-slide panel but render as a tiny floating postcard
- slides that read like projected documents instead of presentation surfaces

## Aesthetic Quality

Flag when:
- the stock image feels generic or staged
- the image lowers the precision of the slide
- the deck becomes more decorative than persuasive
- a wow move becomes a gimmick
- visual restraint turns into emptiness or underexplained slides

## Rhythm

Check:
- whether image-led slides are used sparingly
- whether they create a real reset in pace
- whether the deck returns to clean text-led slides after an image moment
- whether the deck alternates between visible proof and spoken depth instead of hiding all depth in notes

## Fix Order

1. Fix syntax leakage
2. Fix broken composition
3. Remove weak images before adding new ones
4. Re-test the slide in context of the neighboring slides
5. If the content still feels wrong, try a different layout intent before rewriting the copy
